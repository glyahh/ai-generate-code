## What would you like to be added?

When a free-tier (Qwen OAuth) user reaches their daily free quota limit, the error message should guide them to upgrade to the **Aliyun Bailian Coding Plan** instead of showing a generic dead-end message.

### Current behavior

When the free daily quota is exhausted, the user sees:

> `Qwen API quota exceeded: Your Qwen API quota has been exhausted. Please wait for your quota to reset.`

This gives the user no actionable next step other than waiting.

### Expected behavior

The error message should:

1. **Clearly state** that the free daily quota has been reached
2. **Recommend upgrading** to the Bailian Coding Plan with direct links to the subscription portal
3. **Tell users how to set it up** — run `/auth`, select "API-KEY", then choose a Coding Plan region
4. **Show the correct portal link**based on region:

   - China: `https://help.aliyun.com/zh/model-studio/coding-plan`
   - Global/Intl: `https://www.alibabacloud.com/help/en/model-studio/coding-plan`

Example message (for reference, exact wording TBD):

```
Your free daily quota has been reached.

To continue using Qwen Code without waiting, upgrade to the Aliyun Bailian Coding Plan:
  China:       https://help.aliyun.com/zh/model-studio/coding-plan
  Global/Intl: https://www.alibabacloud.com/help/en/model-studio/coding-plan

After subscribing, run /auth to configure your Coding Plan API key.
```

## Why is this needed?

- **User retention**: Free-tier users who hit the daily limit currently see a dead-end error with no way forward. Many will stop using the tool rather than discovering the paid plan on their own.
- **Discoverability**: The Bailian Coding Plan already exists and is fully integrated into the `/auth` flow, but it is never surfaced when users actually need it most — at the moment they run out of free quota.
- **Better UX**: A clear, actionable error message with direct links reduces friction. Users can upgrade and resume work in minutes instead of waiting for the daily quota to reset.

## Additional context

- The free quota is enforced server-side by the DashScope API; the client already detects these errors (matching keywords like `insufficient_quota`, `free allocated quota exceeded`)
- The Coding Plan is already fully integrated in the `/auth` → "API-KEY" → "Coding Plan" flow, including region selection (China / Global/Intl) and API key input
- The Coding Plan portal URLs are already defined in the codebase (`ApiKeyInput.tsx`)
- When a quota error is detected for Qwen OAuth users, the request is not retried — it throws immediately, which is the right place to surface the upgrade guidance
