# Competition Best Practices

## Scope

Use this reference for challenge-oriented project decks (e.g., 挑战杯, 三创赛, innovation/entrepreneurship pitches) where the goal is to show clear logic, credible evidence, and execution feasibility.

## 1) Argument before decoration

- Build one main claim for the full deck.
- Make every slide title a conclusion sentence (not a topic label).
- Run a ghost-deck check: titles alone should tell the story.

## 2) Suggested deck architecture (project intro)

1. Cover: project value proposition in one sentence
2. Problem and urgency
3. Existing solutions and gap
4. Your solution and differentiation
5. Core innovation points (technical/business/model)
6. Product or system architecture
7. Validation evidence (pilot, experiment, user test, data)
8. Market and scenario fit
9. Business model and commercialization path
10. Milestones, resources, and execution plan
11. Team and capability match
12. Risk and mitigation
13. Closing ask / expected support
14. Appendix/Q&A backup slides

## 3) Exhibit discipline

- One primary exhibit per slide (chart/table/diagram/equation).
- Explain the key takeaway directly on the exhibit.
- Prefer readable charts over dense tables.
- Remove visual elements that do not support the argument.

## 4) Text and typography discipline

- Keep body text concise; avoid paragraph blocks.
- Keep one sans-serif family and use weight/size for hierarchy.
- Keep body text left-aligned for readability.
- Keep citations and sources visible when using external facts or figures.

## 5) "High-end, low-AI-signature" checklist

- Avoid common AI visual artifacts: random gradients, decorative icons, mixed visual styles.
- Keep color system tight: neutral base + one accent + one alert/emphasis color.
- Keep spacing rhythm consistent across all slides.
- Prefer understated refinement over visual novelty.
- Emphasize authentic project evidence over polished-but-empty aesthetics.

## 6) Timing and Q&A preparedness

- Plan approximately one content slide per minute.
- Keep 1-2 minutes buffer in rehearsal.
- Prepare appendix slides for likely judge questions:
  - method details
  - cost model assumptions
  - risk sensitivity
  - compliance/IP/implementation limits
